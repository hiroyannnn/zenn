## これは何か

`vscode-github-issues-sync` は、GitHub Issues を VS Code からローカルに同期して、YAMLフロントマター付きMarkdownとして保存する拡張機能です。AIコードエージェントにIssueを渡したいときや、オフラインで参照したいときに便利です。

<https://github.com/hiroyannnn/vscode-github-issues-sync>

## 背景

IssueはWeb上で読むのが前提ですが、AIに渡すときは「ローカルにまとまっていること」「Markdownで構造化されていること」が効きます。  
また、移動中やネットワークが不安定な環境でもIssueを参照したいケースがあります。

この拡張は、Issueをローカルに同期してファイル化することで、そのまま検索・参照・AI入力に使える状態を作ります。

## インストール

### VS Code Marketplace（公開後）

1. VS Codeを開く
2. 拡張機能ビュー（`Ctrl+Shift+X` / `Cmd+Shift+X`）を開く
3. "GitHub Issues Sync"を検索
4. "Install"をクリック

### 開発版のインストール

```bash
git clone https://github.com/hiroyannnn/vscode-github-issues-sync.git
cd vscode-github-issues-sync
npm install
npm run compile
```

F5キーでExtension Development Hostを起動します。

## 基本的な使い方

### 初期設定

1. GitHubリポジトリのワークスペースを開く
2. コマンドパレット（`Ctrl+Shift+P` / `Cmd+Shift+P`）を開く
3. "GitHub Issues Sync: Configure"を実行

### 手動同期

コマンドパレットから "GitHub Issues Sync: Sync Now" を実行します。進捗通知と結果通知が出ます。

### Tree Viewでの閲覧

アクティビティバーの "GitHub Issues" からIssue一覧を開けます。クリックすると同期済みのMarkdownが開きます。

## 同期されるファイル形式

Issueはデフォルトで `.vscode/github-issues/` に保存されます。

```
.vscode/github-issues/
├── issue-1.md
├── issue-2.md
└── ...
```

ファイルはYAMLフロントマター＋本文Markdownです。

```markdown
---
id: 123
number: 456
title: "Issue Title"
state: open
created_at: "2024-01-01T00:00:00Z"
updated_at: "2024-01-02T00:00:00Z"
author:
  login: username
  id: 12345
labels:
  - bug
  - enhancement
---

# Issue Title

Issue本文がここに表示されます。

## Comments

### @commenter1

コメント内容...
```

## 同期戦略とフィルタ

大きいリポジトリでも扱いやすいよう、同期戦略とフィルタが用意されています。

| 設定 | 役割 | 補足 |
|------|------|------|
| `githubIssuesSync.syncStrategy` | 同期方法 | `full` / `incremental` / `lazy` |
| `githubIssuesSync.syncPeriod` | 期間 | `3months` / `6months` / `1year` / `all` |
| `githubIssuesSync.labelFilter` | ラベル絞り込み | 空配列で全ラベル |
| `githubIssuesSync.milestoneFilter` | マイルストーン絞り込み | 空配列で全マイルストーン |
| `githubIssuesSync.maxIssues` | 同期上限 | デフォルト100 |

`lazy` はメタデータだけ同期し、必要時に詳細を取得するモードです。大量のIssueがある場合に有効です。

## 認証

認証は以下の順序で試行されます。

1. VS Code標準のGitHub認証（推奨）
2. 設定 `githubIssuesSync.personalAccessToken`
3. SecretStorageに保存されたPAT

PATを使う場合のスコープは `repo`（プライベート）または `public_repo`（パブリック）です。

## AI/LLMへの渡し方

同期後はローカルファイルなので、検索や抽出が簡単です。

```bash
rg "auth" .vscode/github-issues
```

そのままLLMに渡す場合は、該当IssueのMarkdownを入力にします。

```bash
cat .vscode/github-issues/issue-123.md | claude "このIssueに対応する修正案を考えてください"
```

## 注意点

- ワークスペースがGitHubリポジトリのルートである必要があります
- GitHub APIのRate Limitに注意してください
- 自動同期は最短5分間隔です（`syncInterval`）

## まとめ

`vscode-github-issues-sync` は、Issueをローカルに同期してMarkdown化するVS Code拡張です。

- Issueをファイル化してオフライン参照・検索ができる
- YAMLフロントマター付きでAIに渡しやすい
- 自動同期・フィルタ・同期戦略で大規模リポジトリにも対応

IssueをAIと一緒に扱うワークフローの土台として使ってみてください。
